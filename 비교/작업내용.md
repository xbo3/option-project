# OPTION 프로젝트 작업내용 정리
## 최종 업데이트: 2026-03-07

---

## 1. 프로젝트 개요

**OPTION** — 한국 베팅 시장을 위한 가격비교 커뮤니티 플랫폼
- 기존 배너 중심 커뮤니티를 대체하는 가격비교 + 콘텐츠 중심 접근
- 핵심 차별점: 업체가 광고비가 아닌 실제 보너스 조건으로 경쟁
- 슬로건: "1%만 높아도 1위 노출"

---

## 2. 수익 모델

| 항목 | 내용 |
|------|------|
| 파트너페이지 수수료 | 30% (시장 표준 40%), 15% 유저 이벤트 환원, 15% 운영 |
| 옵션 패키지 | 월 99만원 → 100% 유저 쿠폰 전환 (33매 × 3만원) |
| 보험 상품 | 500만원 보장, 월 보험료 10%, 자발적 가입 |
| 100명에게 물어봐 | 메인 슬라이더 슬롯 시간당 10만원 — 전액 유저 보너스 환원 |
| 업체 등록 조건 | 일일 정산 필수 |

---

## 3. 랭킹 시스템

### 카테고리 (4개)
카지노 / 슬롯 / 스포츠 / ETC

### 서브 카테고리 (각 7개 + 추가 정렬 3개)
- 첫충보너스높은순, 매충보너스높은순, 캐시백보너스높은순
- 올인보너스높은순, 가입보너스높은순, 이벤트많은순, 롤링요율높은순
- (추가) 이용회원많은순, 별점높은순, 후기많은순

### 공동1위 (Tied Ranking) 로직
- 동일 퍼센트 → 둘 다 골드 카드(r1 class) + "공동1위" 뱃지
- 타이브레이커: 별점 → 후기 → 회원수 (정책 확정 대기)

---

## 4. 유저 등급 시스템

### 점수 산출 공식
```
총 배팅액 × 0.40 + 총 입금액 × 0.25 + 이용 횟수 × 0.20 + 활동 기간 × 0.15
```

### 각설이 등급 (Freeloader 제재)
- 입금 없이 이벤트 당첨 2~4회 → 보너스 1/2~1/4만 지급
- 차감분은 실제 이용 회원에게 재분배

### VIP 등급
- 1티어: 접속만 해도 하루 5~10만원 보너스
- 2티어: 매일 특별 이벤트
- 3티어: 이벤트 우선 참여

---

## 5. 별점 시스템

- 총 배팅액 기반 색상 농도 가중치
  - 3천만원 유저의 10점 = 밝은 골드
  - 3만원 유저의 10점 = 거의 투명
- 90일 롤링: 오래된 리뷰부터 만료

---

## 6. 보험 시스템

- 보장 금액: 500만원 (프리미엄 1000만원 확장 가능)
- 월 보험료: 보장 금액의 10%
- 최초 신고자: 50% (250만원), 나머지 N등분
- 자발적 가입 → "🛡️ 보험 가입 업체" 뱃지

## 7. UI 디자인 (최종: option-complete.html)

### 헤더
- OPTION 14px 골드 그라디언트, 센터 정렬, 30px 높이

### 하단 글라스모피즘 버튼 (대칭)
- ☰ 좌하단 (56×56 원형) → 카테고리 패널 좌측 슬라이드
- 👤 우하단 (56×56 원형) → 프로필 패널 우측 슬라이드
- 스타일: backdrop-filter blur(24px), rgba(255,255,255,.35), inset highlight

### 좌측 카테고리 패널 (220px)
- 카지노/슬롯/스포츠/ETC 이미지 포함 아코디언
- 각 7개 서브 아이템

### 우측 프로필 패널 (260px)
- 👤 내 정보 / 📝 업체 등록 / 🎁 이벤트 / 🏆 순위 비교

### 정렬 탭 (10개)
첫충 / 매충 / 캐시백 / 올인 / 가입 / 이벤트 / 롤링 / 이용회원많은순 / 별점높은순 / 후기많은순

### 패키지 시스템
- pkg_font: gold/blue/purple/cyan/orange
- 효과: 업체명 색상 + 카드 테두리/그라디언트 틴트

### 샘플 데이터 (10개 업체)
겜블시티(gold), 벳콩(blue), 럭키스타, 크리스탈벳(purple), 플래티넘, 다이아몬드(cyan), 로얄킹, 골드러시(orange), 에이스벳, 비트윈

## 8. 기술 스택 / 배포

### 프로젝트 위치
- 로컬: `C:\Users\woo\option-project\`
- GitHub: `https://github.com/xbo3/option-project.git`
- Railway: `https://confident-alignment-production.up.railway.app`

### 프로젝트 구조
```
option-project/
├── server.js          (Express + multer 이미지 업로드 API)
├── package.json       (start: node server.js)
├── .gitignore
├── data.json          (사이트 데이터 저장소)
└── public/
    ├── index.html     (메인 UI — option-complete.html)
    └── uploads/       (업로드 이미지)
```

### API 엔드포인트
- `GET /api/sites` — 전체 사이트 목록
- `POST /api/sites` — 사이트 등록 (multipart/form-data + 이미지)
- `DELETE /api/sites/:id` — 사이트 삭제

### 배포 플로우
1. 로컬에서 코드 수정
2. `git add -A && git commit -m "메시지" && git push origin master`
3. Railway 자동 배포 (GitHub 연동)
